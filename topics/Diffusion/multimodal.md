## What is text diffusion

**T:** Basic

**A:** ext diffusion is a way of generating text using a diffusion-style denoising process instead of generating one token at a time like an autoregressive language model. In image diffusion, the model starts from random noise and gradually denoises it into an image. For text, the idea is similar, but the “noise” is usually applied to discrete tokens or latent text representations. For example, a model may start with a sequence of masked or corrupted tokens and iteratively refine them into coherent text.

## How are longer context genrated

**T:** Hands on

**A:** Longer contexts can be generated by denoising a longer sequence of tokens or latent representations at once. Instead of predicting only the next token, the model predicts or refines many positions in the sequence over multiple denoising steps. In practice, long-context generation can be handled in a few ways. One approach is to initialize a long sequence with masked or noisy tokens and iteratively fill in the whole sequence. Another approach is chunk-based generation, where the model generates or refines overlapping blocks of text so that local coherence is preserved while still maintaining broader context. Some systems also use hierarchical generation, where the model first creates a high-level plan or structure and then fills in details.

## Why is text diffusion hard

**T:** Deep

**A:** Text diffusion is hard mainly because text is discrete, while diffusion models were originally designed for continuous data like images, audio waveforms, or latent vectors. In images, adding Gaussian noise makes intuitive sense: pixels gradually become noisy, and the model learns to reverse that process. In text, adding small continuous noise to a word token does not directly make sense because tokens are categorical. A token like “cat” cannot be slightly changed into “dog” in the same natural way that a pixel value can be slightly perturbed.

Because of this, text diffusion models usually need special corruption processes. For example, they may replace tokens with {MASK}, randomly substitute tokens, delete tokens, or diffuse in a continuous embedding space rather than directly over raw tokens. Each choice has tradeoffs. Masking is more natural for language but may not behave like standard Gaussian diffusion. Embedding-space diffusion is more continuous, but the model still has to map the final continuous representation back to valid discrete tokens.

Another difficulty is fluency. Autoregressive models are very good at local next-token prediction, which matches the structure of language. Diffusion models need to produce globally coherent text after iterative refinement, which can be more complex and computationally expensive.
	
## What are audio diffusion models, how do they work?

**T:** Basic

**A:**  Audio diffusion models generate audio by starting from noise and gradually denoising it into a realistic waveform or audio representation. They can operate directly on raw waveforms, on spectrograms, or in a compressed latent space. A common approach is to convert audio into a spectrogram, which represents how frequency content changes over time. Since a spectrogram is image-like, many image diffusion ideas can be adapted to audio. The diffusion model learns to generate or denoise this spectrogram, and then another model or algorithm converts it back into waveform audio.

Audio is challenging because it has both temporal structure and frequency structure. A raw waveform is one-dimensional, but it contains complex information about pitch, rhythm, timbre, phase, and frequency. Spectrograms make frequency structure easier to model, but phase information can be hard to recover accurately. That is why many audio systems use specialized representations, neural vocoders, or latent audio spaces.
	
## How are input modalities regularized in multimodal systems

**T:** Hands on

**A:** A common way to regularize multimodal systems is modality dropout, where one or more input modalities are randomly removed during training. For example, a model trained on image, text, and audio may sometimes receive all three, sometimes only image and text, and sometimes only text. This prevents the model from over-relying on a single modality. If the image is missing or noisy, the model should still use text. If the text is vague, the model should still use visual or audio information. This makes the system more robust at test time, especially when some sensors fail or some inputs are unavailable. 

	
## Why is diffusion used in robotics

**T:** Hands on

**A:** Diffusion is useful in robotics because robot behavior is often multimodal. For the same task, there may be many valid trajectories. For example, a robot arm can reach around an obstacle from the left or the right. A standard deterministic policy may average these choices and produce an invalid motion, while a diffusion model can represent multiple possible trajectories. In robotics, diffusion models are often used to generate action sequences or trajectories conditioned on the current observation, goal, language instruction, or environment state. This is similar in spirit to trajectory optimization: instead of choosing only the next action, the model generates a full future action plan and then executes part of it.
	
## How is masking done in diffusion llms?

**T:** Basic

**A:** In diffusion language models, masking can be used as the noising process. Instead of adding Gaussian noise to tokens, the model randomly replaces some tokens with a special [MASK] token or another corrupted symbol. The model then learns to reconstruct the original tokens from the partially masked sequence.During generation, the model may start with a mostly or fully masked sequence and gradually fill in tokens over multiple denoising steps. At each step, it predicts likely tokens for masked positions, then keeps high-confidence predictions and continues refining uncertain positions.
	
## Why diffusion leads to smooth movements in robots

**T:** Hands on

**A:** Diffusion can lead to smooth robot movements because it usually generates an entire action trajectory rather than a single action at each time step. Since the model predicts a sequence of future actions jointly, it can learn temporal consistency across the trajectory. For example, instead of deciding “what should the robot do right now?” independently at every step, the model decides “what sequence of actions should move the robot toward the goal?” This naturally encourages smoother motion because neighboring actions are generated together and conditioned on the same planned behavior.

Diffusion models also learn from demonstration trajectories, which are often smooth because they come from human teleoperation, expert controllers, or trajectory optimization. During denoising, the model gradually refines noisy trajectories toward realistic ones, removing abrupt or inconsistent movements.
	
## How are actions selected in world models

**T:** Deep

**A:** In world models, the agent learns a predictive model of the environment, often in a latent representation. Instead of planning directly in raw pixels or raw sensor space, the world model encodes observations into latent states and predicts how those latent states evolve under different actions. Actions can be selected by imagining possible futures. The agent tries candidate action sequences in the learned latent dynamics model, predicts the resulting future states and rewards, and chooses the action sequence that looks best. Usually, only the first action is executed, then the agent observes the real next state and replans. In some systems, actions are optimized directly using planning methods such as MPC, CEM, MPPI, or gradient-based optimization. In others, a policy is trained inside the learned world model. The core idea is that the model predicts: current latent state plus action gives next latent state, reward, or future observation. The policy or planner then uses those predictions to select actions.
	
## What are the pros and cons of diffusion and auto regressive models

**T:** Deep

**A:** Autoregressive models generate outputs sequentially, one token or action at a time. This works very well for language because text naturally has a left-to-right structure. They are usually simple to train, highly scalable, and very strong at fluent generation. However, once an autoregressive model generates a token, it usually cannot revise it unless special editing mechanisms are used. Generation can also be slow for long outputs because tokens are produced one by one.

Diffusion models generate by iterative refinement. They start from noise or a corrupted sequence and gradually denoise it into a final output. This allows them to model complex multimodal distributions and revise many parts of the output together. In robotics, this is useful because there may be many valid action trajectories. In images and audio, it is useful because global structure can be refined over many steps.

The main downside of diffusion is computational cost. Generation often requires many denoising steps, which can be slower than a single forward pass or a short autoregressive rollout. For text, diffusion is also harder because tokens are discrete, so the noising process is less natural than in images or continuous control.
