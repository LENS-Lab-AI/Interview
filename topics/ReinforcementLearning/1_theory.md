# Theory - Som


### Core Concepts & Foundations
* **What is the Bellman Equation?**
    * It is used to calculate the value of a specific state.
* **What is a Markov Chain?**
    * A system where the next state depends *only* on the current state. No actions are involved. This concept is the foundation of MDPs.
* **What is a Markov Decision Process (MDP)?**
    * An extension of a Markov Chain that includes actions and rewards. The Markov property (next state depends only on the current state and action) is a fair assumption to simplify complex problems.
* **What is a Q-value vs. a Normal (State) Value?**
    * A Normal Value evaluates how good a *state* is. A Q-value evaluates how good a *specific action* is within a given state. 


### Environments & Dynamics
* **Where can we NOT use RL?**
    * In non-stationary environments where the Markov property does not hold true (i.e., the rules of the environment are constantly changing).
* **Model-Based vs. Model-Free RL:**
    * **Model-Based:** The agent has (or learns) a model of the environment's transition probabilities. It can be less accurate due to model errors. *(Examples: MCTS/AlphaZero, MPC, LQR, PILCO)*.
    * **Model-Free:** The agent learns directly from experience without knowing the transition probabilities. It is generally better suited for highly complex environments.
* **Optimal Control vs. Reinforcement Learning:**
    * *(General distinction)* Optimal control typically assumes the mathematical model of the environment is already known, whereas RL generally learns through trial and error.

### Learning Paradigms
* **On-Policy vs. Off-Policy:**
    * **On-Policy:** Learns from the actions taken by the *current* policy it is trying to evaluate or improve.
    * **Off-Policy:** Learns from data/updates generated by a *different* policy (or older data).
* **Online vs. Offline RL:**
    * **Online:** The agent actively interacts with the environment to learn.
    * **Offline:** The agent learns from a static, pre-collected dataset without any live environment interaction.

### Types of RL Algorithms
* **Value-Based:** Learns a value function to figure out the best states/actions, and derives the policy from that.
* **Policy Gradient:** Directly updates and optimizes the policy without relying on a value function *(Example: PPO)*.
* **Actor-Critic:** Combines both methods. The "Actor" updates the policy, and the "Critic" evaluates the actor by learning the value function.

### Core Challenges in RL
* **The Exploration vs. Exploitation Trade-off:**
    * **Exploitation:** Choosing the action that we currently know yields the best reward.
    * **Exploration:** Trying out new, different actions to discover potentially better rewards.
* **What is Credit Assignment?**
    * When an agent receives a reward, it needs to figure out *which specific past actions* influenced that outcome. This is especially difficult with **sparse** or **delayed rewards** (where the reward comes long after the crucial actions were taken).
* **What is Generalization?**
    * The ability of the RL agent to take what it has learned and apply it successfully to unknown or unseen states.
